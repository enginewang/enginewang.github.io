Explain Input and Output Streams in C++

Posted on 2018-10-11 | In Tech

After learning the basic C++ grammar, I still confuse on some input operations, so I tried to write this note, talking about some specific things about istream and ostream in C++.

These pictures shows the relationship between some Stream Classes.

Input Stream

use array form to asist input

Input one item

Input specific number of items


get() and getline() method

None parameter

One parameter

cin.get(c) read a character from istream and assign it to c;

Three parameters

cin.get(array_name, length, 'end_char') (nums means will read n-1 chars)

just remember: it will end if(reach the length||find the endchar)


getline() method

getline() can read a line of input chars, similar to get(ch, num, 'end_char')

diffrence between getline() and cin

  • getline() can read blank characters, but cin can’t, it will end reading when meeting blanks
  • getlin() can only read chars, while cin can read every types.


Other methods:

eof()

eof means end of file, we usually use it to judge the end.


peek() and putback()

peek() will return the next char(only observe it, pointer will not move)

putback() will insert specific char to the pointer before the current char

★ a comprehensive example:


ignore()

the function of ignore() is to skip some chars
cin.ignore(length, 'end_char')

e.g.
if we add cin.ignore(4, 'd'); before the second getline() in the previous example, the second part will be "ding and playi"


String stream

stringstream is such a useful method, aimming at strings.
You should include <sstream> before using it.

convert data type



Output Stream

Ostream define three objects for outputting:

cout cerr clog
standard output stream standard error stream standard error stream


cerr

the information in cerr object can only display on the screen, it won’t affact the memory file.
cerr is always used to output error information quickly.

clog

information in cerr won’t go through buffer, but clog information will be put in, when the buffer is full or meet endl, clog information will be displayed.

cout

cout has some useful function to control the output formats:

functions description
setprecision(n) set precision of nums
setw(n) set width
setfill(c) set the filling char
setioflags() set format
resetioflags() reset format

other functions will be talked about later.
